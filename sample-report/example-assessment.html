Security Best Practices
Recommendations beyond this assessment — the security journey for any M365 tenant
Enable Multi-Factor Authentication for all users
MFA is the single most impactful control — blocks 99.9% of automated attacks. Start with Authenticator app, enforce via CA policy.
Quick Win (days)
Block legacy authentication protocols
Legacy auth cannot be protected by MFA. A single legacy auth login from any user bypasses all CA policies. Block via CA or disable org-wide.
Quick Win (days)
Configure Conditional Access baseline policies
CA is the policy engine for Zero Trust. Minimum: require MFA for all users, block legacy auth, require compliant device for sensitive data.
Medium (weeks)
Review and offboard stale accounts
Implement an offboarding checklist: disable account, revoke sessions, remove licenses, review forwarding rules, archive mailbox. Run quarterly audits.
Quick Win (days)
Deploy Microsoft Defender for Endpoint
Endpoint visibility is the foundation of incident response. Without EDR, endpoint threats are invisible until data is gone.
Medium (weeks)
Implement DMARC at enforcement (p=reject)
SPF and DKIM alone do not prevent spoofing. Only p=reject tells receiving servers to block messages that fail authentication. Non-trivial but critical for any client-facing domain.
Medium (weeks)
Establish PAM with PIM for privileged access
Permanent admin accounts are always-on targets. PIM elevates admins on demand with MFA, justification, and time limits — significantly reducing the blast radius of a compromise.
Strategic (months)
Deploy sensitivity labels and DLP policies
Data protection requires knowing what data you have. Labels + DLP provides classification, protection, and enforcement across email, SharePoint, Teams, and endpoints.
Strategic (months)
Create and test an incident response plan
When a BEC or ransomware incident occurs, decisions made in the first 30 minutes determine the outcome. Having a playbook, a contact list, and a tested process cuts response time dramatically.
